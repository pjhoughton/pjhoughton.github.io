Glossary

TermExplanation
ImageA lightweight, stand-alone package that includes everything needed to run software.
ContainerA running instance of an image, isolated from the host system.
DockerfileA text file with instructions to build a Docker image.
VolumePersistent storage that containers can use to save data.
NetworkA way for containers to communicate with each other.
RegistryA storage and distribution system for Docker images (e.g., Docker Hub).
TagA label applied to an image to identify its version.
ComposeA tool for defining and running multi-container Docker applications using YAML files.
ServiceA container definition in a Compose file, e.g., web, db.
PruneClean up unused Docker objects (containers, images, volumes, networks).
OrphanA container left over from a previous Compose run that is no longer defined.
