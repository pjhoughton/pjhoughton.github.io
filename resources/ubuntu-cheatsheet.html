Glossary

TermExplanation
Distro (Distribution)A Linux version built on the Linux kernel (Ubuntu, Debian, Fedora, etc).
KernelThe core of the OS that manages hardware, memory, processes, and drivers.
Package ManagerSoftware that installs, updates, and removes packages. Ubuntu uses APT.
APTAdvanced Package Tool — Ubuntu’s primary package manager.
RepositoryOnline storage of software packages for APT to install from.
Daemon (Service)A background process that runs continuously (e.g., SSH, Apache).
SystemdUbuntu’s service manager. Controls services, logs, startup behavior.
ShellThe text-based interface for running commands (bash, sh, zsh).
BashUbuntu’s default shell environment for command execution.
RootThe superuser account with full system access.
SudoAllows a normal user to run commands with root privileges.
Home DirectoryA user's personal area (/home/username).
File PermissionsRead, write, and execute settings for users, groups, and others.
ProcessA running program or task on the system.
PackageA compressed bundle of software files for installation.
ServiceA managed background program controlled by systemctl.
UFWUncomplicated Firewall — Ubuntu’s simple firewall manager.
SSHSecure Shell — used for remote login to servers.
SwapDisk space used as overflow memory when RAM is full.
MountAttaching a filesystem so it is accessible under a directory path.
SyslogSystem log file containing general system events.
JournalctlSystemd’s logging system for viewing service logs.
Environment VariableA configurable value that affects how commands and programs run.
