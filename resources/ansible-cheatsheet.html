Ansible Cheatsheet

Basic Commands

DescriptionCommand
Ping all hostsansible all -m ping
Run command on hostsansible all -a "uptime"
Use inventory fileansible -i inventory.ini all -m ping
Run playbookansible-playbook site.yml
Check syntaxansible-playbook site.yml --syntax-check

Inventory & Variables

DescriptionCommand

host1 ansible_user=admin

Useful Modules

DescriptionCommand
Copy fileansible all -m copy -a "src=/tmp/file dest=/etc/file"
Install packageansible all -m apt -a "name=nginx state=present"
Start serviceansible all -m service -a "name=nginx state=started"

Glossary

TermExplanation
AnsibleAn automation tool for configuration management, app deployment, and orchestration.
PlaybookA YAML file containing tasks that define automation workflows.
TaskAn Ansible action that uses a module to perform work.
ModuleThe unit of work in Ansible (e.g., copy, apt, service).
InventoryA list of managed hosts; can be static or dynamic.
Host GroupA named collection of hosts in the inventory.
FactsAutomatically gathered system information (OS, memory, network, etc.).
Variables (Vars)Data values used in templates, playbooks, and tasks.
HandlersSpecial tasks triggered only when notified (useful for restarting services).
RoleA structured way of organising playbooks (tasks, templates, vars, files).
YAMLHuman-readable structured format used for playbooks.
IdempotentProperty of Ansible tasks where running them multiple times results in no change.
Control NodeThe machine running Ansible.
Managed NodeThe servers Ansible configures.
ansible.cfgMain configuration file that controls Ansible behaviour.
Ad-Hoc CommandA single Ansible command run without a playbook.
Callback PluginCustomises output and logging.
Filter PluginExtends Jinja2 templating used in playbooks.
TemplateA Jinja2-based file that supports variables and logic.
SSH KeysRecommended authentication method for Ansible connections.
